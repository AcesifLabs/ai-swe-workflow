---
description: Generate execution plan and task files based on repo conventions
---

Please analyze the task `$ARGUMENTS` and prepare a detailed execution plan with task files.

## Step 1: Find and read repo instructions file

Look for the following files in the project repository root (in order of precedence):
- AGENTS.md
- CLAUDE.md
- GEMINI.md

Read the first file found to understand the repo's rules, conventions, do-s, and don't-s.

## Step 2: Explore the codebase structure

Explore the codebase to understand:
- Project structure and architecture
- Key directories and their purposes
- Configuration files (package.json, pyproject.toml, etc.)
- Existing patterns and conventions
- Testing setup and patterns
- Build/lint/typecheck commands

## Step 3: Analyze the task

Break down the task into smaller, actionable steps considering:
- The repo's conventions and rules from Step 1
- The codebase structure from Step 2
- Dependencies and prerequisites
- Potential risks or blockers

## Step 4: Ask clarifying questions (if needed)

Before generating tasks, conduct a thorough interview with the user to extract ALL requirements and constraints:

**Critical questions to ask:**
- What is the exact scope and boundary of this work?
- Are there any specific files, modules, or areas that should NOT be modified?
- What are the success criteria beyond "it works"?
- What dependencies or external systems are involved?
- What testing approach is expected (unit tests, integration tests, manual testing)?

Note: Coding style, architectural patterns, and conventions are already covered in the repo's AGENTS.md (read in Step 1).

**Confidence Check:**
After asking questions, the agent must self-assess:
- "Do I have a clear, unambiguous understanding of what needs to be built?"
- "Are all edge cases and boundary conditions identified?"
- "Do I know exactly what files will be affected?"
- "Is the success criteria objectively verifiable?"

**Iterate Until Confident:**
If ANY vagueness exists, the agent MUST:
1. Ask follow-up questions
2. Restate understanding and ask for confirmation
3. Only proceed to Step 5 when 100% confident no ambiguity remains

**IMPORTANT: Planning Mode Only**
The agent is in SPECIFICATION MODE ONLY and should NOT:
- Write any code
- Modify any files
- Create any implementation artifacts

The agent should explicitly state: "I have all the information I need to generate precise, unambiguous tasks" before moving forward.

## Step 5: Determine output format and identifier

1. **Ask user** for their preference on output format:
   - Single file: `tasks.md` with all tasks
   - Folder mode: `tasks-<identifier>/` with `task-xxx.md` files + `progress.md`

2. **Default to folder mode** if user doesn't specify

3. **Generate unique identifier**:
   - Use current git branch name (sanitized) if available
   - Otherwise, create concise identifier from task description (first 3-4 words, alphanumeric)

## Step 6: Generate detailed execution plan

Output a structured plan with:
1. **Prerequisites/Pre-checks**: What needs to be verified before starting
2. **Phase-by-phase breakdown**: Clear steps with descriptions
3. **File locations**: Which files will be affected
4. **Testing approach**: How to verify the changes
5. **Potential issues**: Known risks and mitigation strategies

## Step 7: Generate task files

Based on user preference from Step 5:

**Single file mode (`tasks.md`):**
```markdown
# Tasks

## Task 1: [Title]

Goal: `<verification-command>`

Description:
[Brief description of what needs to be done]

## Task 2: [Title]
...

---

## Progress
- Task 1: pending
- Task 2: pending
```

**Folder mode (`tasks-<identifier>/`):**
- Create folder `tasks-<identifier>/`
- Generate `task-001-[slug].md` for each task:
  ```markdown
  # Task [N]: [Title]
  
  Goal: `<verification-command>`
  
  Description:
  [Detailed description]
  
  Notes:
  - [Any additional notes]
  ```
- Generate `progress.md`:
  ```markdown
  # Task Progress
  
  ## Overview
  - Total Tasks: [N]
  - Completed: 0
  
  ## Tasks
  - [ ] Task 1: [Title] (pending)
  - [ ] Task 2: [Title] (pending)
  
  ## Current Task: 1
  ```

The plan should be detailed enough to be actionable and follow the repo's established patterns.

## Step 8: Delete tasks-<identifier>/ directory and progress.md

Once you are done completing all the tasks, remove any markdown files with the extension .md that were generated by the agent for the task completion.
